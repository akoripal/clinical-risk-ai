# Clinical Risk AI — Agentic Hospital Utilization Assistant

An end-to-end AI engineering project that transforms a healthcare analytics 
workflow into an intelligent clinical decision support system. Built on 101,766 
diabetic inpatient encounters from the UCI Diabetes 130-US Hospitals dataset.

---

## What This Project Does

A care coordinator opens the app, configures a patient profile, and receives:
- A **prolonged stay risk probability** from a trained ML model
- A **risk tier** (Low / Moderate / High) for operational triage
- A **clinical narrative** generated by Llama 3.3 explaining the key drivers 
  and recommending a specific next step
- A **conversational interface** to ask follow-up questions about the patient

---

## Architecture
```
Raw Data (UCI Diabetes Dataset)
        ↓
Data Pipeline (pandas + feature engineering)
        ↓
Risk Models (Logistic Regression · XGBoost)
        ↓
FastAPI Endpoint  ←→  Groq / Llama-3.3 (clinical narrative)
        ↓
Streamlit Chat UI (conversational risk assistant)
```

## Model Performance

| Model | AUC | Notes |
|---|---|---|
| Logistic Regression | 0.7498 | Primary model — interpretable, auditable |
| XGBoost | 0.7712 | Challenger model — higher accuracy |

Trained on 55,978 encounters, evaluated on 13,995. Class imbalance (14% 
prolonged stay rate) handled with SMOTE oversampling. A data leakage issue 
(`time_in_hospital` directly encodes the target) was identified and resolved 
during development.

## Quickstart
```bash
# 1. Clone and set up environment
git clone https://github.com/YOUR_USERNAME/clinical-risk-ai.git
cd clinical-risk-ai
/opt/homebrew/opt/python@3.11/bin/python3.11 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 2. Add your Groq API key
echo "GROQ_API_KEY=your-key-here" > .env

# 3. Download and prepare data
python src/pipeline/download_data.py
python src/pipeline/preprocess.py

# 4. Train models
python src/model/train.py

# 5. Start API (terminal 1)
uvicorn src.api.main:app --reload

# 6. Start UI (terminal 2)
streamlit run src/ui/app.py
```

## Key Engineering Decisions

**Why logistic regression as the primary model?** Clinical interpretability 
is a requirement in healthcare AI. Logistic regression produces auditable 
coefficients that clinicians can reason about. XGBoost is included as a 
challenger model to quantify the accuracy/interpretability tradeoff.

**Why separate the ML model from the LLM?** The model scores risk. The LLM 
explains it. Mixing these responsibilities — asking an LLM to directly predict 
clinical outcomes — would introduce hallucination risk on safety-critical 
outputs.

**Why Groq + Llama-3.3?** Inference speed matters in a clinical workflow. 
Groq's hardware acceleration delivers sub-second narrative generation, making 
the tool viable for point-of-care use.
